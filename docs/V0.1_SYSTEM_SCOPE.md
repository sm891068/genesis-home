# v0.1 系統範圍

本文件定義 `v0.1 Rebuild Foundation` 需要建立的系統邊界。

## 主選單

固定 8 個入口：

1. 新的傳承
2. 繼續遊戲
3. 父親主線
4. 幫派基地
5. 聯機戰鬥
6. 夥伴圖鑑
7. 榮耀殿堂
8. 設定

## 存檔與讀取

v0.1 要定義一份乾淨的新存檔格式。

建議欄位：

```json
{
  "version": "0.1",
  "playerName": "",
  "gangName": "",
  "day": 1,
  "resources": {
    "money": 0,
    "reputation": 0,
    "manpower": 0,
    "heat": 0
  },
  "routes": {
    "honor": 0,
    "capital": 0,
    "charm": 0,
    "force": 0,
    "strategy": 0
  },
  "story": {
    "fatherChapter": "prologue",
    "unlockedClues": []
  },
  "companions": [],
  "relationships": {}
}
```

## 角色資料

角色資料需要支援後續擴充。

建議欄位：

- ID。
- 名稱。
- 類型：女王、秘書、戰鬥、情報、醫療、支援。
- 稀有度。
- 性格。
- 能力值。
- 技能。
- 信任度。
- AI 記憶。
- 是否可進入後宮/信任劇情。

## AI 互動

AI 互動不直接等於真正大型模型連線。v0.1 先定義：

- 玩家輸入。
- 角色性格。
- 信任度。
- 最近事件。
- 角色記憶。
- 回應語氣。
- 關係變化。

## 素材管理

建議資料夾：

```text
assets/
  characters/
  backgrounds/
  ui/
  prompts/
  generated/
```

每個生成素材都應該保留：

- 圖片。
- 角色或場景用途。
- 提示詞。
- 生成日期。
- 是否已進遊戲。

## 聯機戰鬥

v0.1 只建立入口與資料概念。

資料概念：

- 玩家幫派檔案。
- 攻擊隊伍。
- 防守隊伍。
- 戰報。
- 冷卻時間。
- 排行榜預留。

## v0.1 完成定義

v0.1 完成時，不要求遊戲完整可玩，但要讓下一步開發有清楚地基：

- 版本方向一致。
- 文件一致。
- 主選單入口一致。
- 存檔格式明確。
- 角色與 AI 互動資料結構明確。
- 素材如何放進 repo 有規則。
