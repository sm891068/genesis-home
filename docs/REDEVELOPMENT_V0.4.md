# v0.4.0 Redevelopment 重新開發規劃

本版本目標是把 `黑道建築 Underworld Architect` 從展示型原型調整成可長期開發的遊戲骨架。v0.4.0 不追求一次完成所有玩法，而是先完成三個核心方向：主選單重構、聯機戰鬥規格、父親主線大綱。

## 版本定位

v0.4.0 是重新開發版的起點。

這一版要回答三件事：

1. 玩家進入遊戲後，有哪 8 個穩定功能入口？
2. 玩家之間的 PVP 與幫派攻擊，資料和流程要怎麼設計？
3. 父親留下的故事，如何成為初期主線與玩家動機？

## 優先順序

### 第一優先：主選單重構

主選單是玩家進入遊戲的門面，也會決定後續功能如何擴充。先把 8 個功能入口固定，之後每個入口都可以獨立開發。

主要任務：

- 改為 8 功能主選單。
- 每個功能都有明確名稱、用途與狀態。
- 尚未完成的功能顯示「開發中」，但不讓玩家迷路。
- 主選單視覺保持黑金、雨夜、繼承感。

### 第二優先：父親主線大綱

父親故事是遊戲的情緒核心。玩家不是普通創幫，而是接手一個留下秘密、債務、盟友與敵人的舊勢力。

主要任務：

- 擴寫父親遺言。
- 建立序章與第一章主線。
- 設定父親的盟友、叛徒、舊敵與遺產。
- 把五條路線連到父親當年的選擇。

### 第三優先：聯機戰鬥規格

聯機戰鬥是長期玩法，但不應該第一天就硬做完整伺服器。先定義 PVP 資料、攻擊流程、戰報與防守隊伍。

主要任務：

- 設計玩家幫派檔案。
- 設計攻擊、防守、戰報流程。
- 設計排行榜和冷卻時間。
- 先用本地模擬對手，之後再接後端。

## 重新開發階段

### Phase 1：文件與主選單

- 完成 8 功能主選單規格。
- 完成父親主線大綱。
- 完成 PVP 系統規格。
- 重構 `public/index.html` 主選單。

### Phase 2：單機核心可玩

- 加入幫派狀態資料。
- 加入 Day 推進。
- 加入基地與資源。
- 加入父親主線序章。

### Phase 3：PVP 模擬

- 加入本地假玩家資料。
- 可攻擊假玩家幫派。
- 產生本地戰報。
- 加入攻擊冷卻與每日次數。

### Phase 4：正式聯機準備

- 設計帳號識別。
- 設計遠端儲存。
- 設計戰鬥結果驗證。
- 設計排行榜同步。

## 設計原則

- 黑道題材用策略、權力與代價呈現，不提供現實犯罪教學。
- 先做可理解的功能入口，再做複雜系統。
- PVP 先模擬，再接後端。
- 父親主線必須推動玩家選擇，而不只是背景故事。
- 所有長期開發都以 GitHub repository 為主。
